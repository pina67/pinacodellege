Marcos de trabajo Java

la palabra inglesa "FRAMEWORK"(marco de trabajo) define, en terminos generales, un conjunto entandarizado de conceptos, practicas y criterios para enfocar un tipo de problematica particular que sirve como referencia, para enfrentar y resolver nuevos problemas de indole similar.

en el desarrolo de software, un framework o infrestructura digital, es una estructura conceptual y tecnologica de soporte definido, normalmente con artefactos o modulos de software concretos, que puede servir de base para la organizacion y desarollo de software. Tipicamente hablando, puede incluir soporte de programas, bibliotecas, y un lenguaje interpretado, entre otras herramientas, para asi ayudar a desarrollar y unir componentes de un proyecto.

Apache Cocoon usualmente llamado simplemente Cocon, es un framework para aplicacion web construido en torno a los conceptos de pipeline, encapsulacion y desarollo de aplicacion web basado en componentes. permite la rapida publicacion de contenidos e una variable de formatos, entre los que se incluyen HTML,PDF y WML.
Apache Struts Struts se basa en el patron de arquitectura de software Modelo-vista.controlador(MVC) el cual se utiliza apliamente y es considerado de gran solidez. El procedimiento se separa en tres secciones diferenciadas llamadas ell modelo, las vistas y elcontrolador.
Google Guice es un framework de inyeccion de independencias que pueden ser utilizado en app hechas con java donde la relacion o dependencia entre objetos de negocio necesita ser asministrada o mantenida
JADE I Java Agent Development framework o JADE es una plataforma software de desarrollo de agentes, inplementada en JAVA
Java pesistence API(JPA) es un framework del lenguaje de programacion Java que maneja datos relacionadas en aplicaciones usando la plataforma java en sus ediciones Standart (Java SE) y enterprise (Java EE). el objetivo que persigue el diseño de esta API es no perder las ventajas de la orientacion a objetos al interactuar con una base de datos
